# Matthew Ouellette – Resume Site

A Hugo-based site that renders your resume from Markdown with multiple designs and downloadable Word/PDF exports.

## Prerequisites

- [Hugo](https://gohugo.io/installation/): `brew install hugo`
- [Pandoc](https://pandoc.org/) (for Word/PDF): `brew install pandoc`
- For PDF export: install 
```shell
brew install --cask basictex
brew install weasyprint
```

## Development

```bash
cd resume-site
hugo server -D
```

Open http://localhost:1313 to view the resume.

## Build for production (with Word & PDF)

```bash
make all
```

This will:
1. Generate `resume.docx` (Word) and `resume.pdf` (PDF) from the Markdown
2. Copy them into `static/` for serving
3. Build the Hugo site into `public/`

Deploy the `public/` folder to any static host (Netlify, Vercel, GitHub Pages).

## Design variants

The site offers three designs, selectable via the toolbar:

- **Professional** – Clean, modern (default)
- **Minimal** – Maximum whitespace, understated
- **Traditional** – Classic, formal, corporate-ready

Use `?design=minimal` or `?design=traditional` in the URL, or click the design links in the toolbar.

## PDF export (print)

To create a PDF that matches the web design:

1. Run `hugo server` and open the site
2. Choose your preferred design
3. Use Print (Cmd/Ctrl + P) → **Save as PDF**

## Project structure

```
resume-site/
├── Makefile             # Builds site + generates Word/PDF
├── config.toml          # Site configuration
├── content/
│   └── _index.md        # Resume content (homepage)
├── layouts/
│   ├── _default/
│   │   └── baseof.html  # Base template (toolbar, design switcher)
│   └── index.html       # Homepage layout
├── static/
│   ├── css/
│   │   ├── resume.css           # Professional design
│   │   ├── resume-minimal.css   # Minimal design
│   │   └── resume-traditional.css
│   ├── resume.pdf       # Generated by make
│   └── resume.docx      # Generated by make
└── public/              # Hugo output (deploy this)
```

## Updating the resume

Edit `content/_index.md`. Then run `make all` to rebuild the site and regenerate Word/PDF exports.
